# **依頼内容**

PySide6とPyMuPDF（fitz）を使用した、MVP（Minimum Viable Product）レベルのPDF結合GUIアプリケーションを作成してください。

# **技術スタック**

- Python 3.13+
- GUIフレームワーク: PySide6
- PDF操作ライブラリ: PyMuPDF (fitz)

# **必須機能（MVP）**

1. ファイルのドラッグ＆ドロップ対応リスト（PDFファイルのみ受け入れ）
2. リスト内ファイルの順序入れ替え（上へ/下へボタン、またはドラッグ）
3. 選択したファイルのリストからの削除
4. 保存先を指定しての結合実行（QFileDialogを使用）
5. 処理完了・エラー時のユーザーへのフィードバック（メッセージボックスなど）

# **拡張性を意識した設計ルール（重要）**

1. **ロジックの分離**: PDFの結合処理を行うクラス（例: PDFService）と、GUIを管理するクラス（例: MainWindow）を完全に分けてください。GUIクラスが直接PyMuPDFのAPIを叩かないようにしてください。
2. **モジュール化**: 今後、ページ抽出や回転などの機能を追加しやすいよう、PDFServiceクラスには各操作をメソッドとして定義してください。
3. **型ヒント**: すべての関数・メソッドにPythonの型ヒント（Type Hits）を付与してください。
4. **非同期処理**: QThreadを使用して、重い処理を並行処理で実行できるように設計してください。ただし、GUIの更新はメインスレッドで行うようにしてください。また、処理中にユーザーがアプリケーションを終了しようとしたら、警告をだしつつ、終了する場合は適切にスレッドを停止してから終了するようにしてください。
5. **テストの作成**: model, view, presenterの各層に対して、単体・結合・回帰テストを作成してください。特にPDFServiceクラスのロジック部分は重点的にテストしてください。

# **コードスタイル**

- PEP 8に準拠
- クラス、メソッドにはDocstring及びインラインコメントを記述。自己文書化を目指す。
- **コード内のコメントおよびGUIの表示文字列（ボタン名やメッセージ）は日本語**とし、変数名やクラス名は英語としてください。

# **出力形式**

- このプロジェクトの他の機能に倣って、MVP的なモジュールの責務分離を実施してください。
