# PDF Toolbox 取扱説明書（非開発者向け）

この文書は、PDF Toolbox の基本操作と、PDF→JPEG 機能の使い方を説明します。

---

## 0. はじめに

### 想定する利用シーン

- PDFを章ごとに分割したい
- 複数PDFを1つにまとめたい
- PDFをまとめて圧縮したい
- 1つのPDFを全ページ JPEG に変換したい

### この文書で分かること

- 起動から各機能の基本導線
- PDF→JPEG の具体的な操作手順
- 出力ルール、制約、上書き確認の意味
- エラーが出たときの対処

## 1. このアプリでできること

- PDF分割: ページを区切って複数PDFとして保存できます
- PDF結合: 複数PDFを並び替えて1つにまとめられます
- PDF圧縮: PDF / フォルダ / ZIP を対象にバッチ圧縮できます
- PDF→JPEG: 単一PDFの全ページを JPEG として一括出力できます

## 2. 起動方法

### 2.1 動作条件

- 対応OS: Windows（本ワークスペースの検証環境）
- Python実行時: `pyproject.toml` 設定は `>=3.14`
- 主な入力形式: `.pdf`

### 2.2 ソース実行（開発環境がある場合）

1. Pythonを準備します（`pyproject.toml` 設定: `>=3.14`）
2. （推奨）仮想環境を有効化します
3. 依存関係をインストールします

   ```bash
   pip install -U pyside6 pillow pymupdf pikepdf
   ```

4. アプリを起動します

   ```bash
   python main.py
   ```

5. 起動時表示を確認します

   起動直後にスプラッシュが表示され、完了後にホーム画面へ切り替わります。

### 2.3 実行ファイル配布版（.exe など）

- 配布された実行ファイルをダブルクリックして起動します
- 初回起動時は SmartScreen 警告が出る可能性があります
- ホーム画面から使いたい機能カードを選択します

## 3. ホーム画面の見方

ホーム画面には、次の機能カードがあります。

- PDF 分割
- PDF 結合
- PDF 圧縮
- PDF → JPEG

準備中の機能カードは押せません。利用可能なカードを押すと、対応する画面へ移動します。

## 4. PDF→JPEG の基本操作

### 4.1 どんなときに使うか

- 1つのPDFをページ画像として配布したいとき
- 各ページを JPEG にして他ツールへ渡したいとき
- PDFの中身をページ単位で画像確認したいとき

### 4.2 画面の見方

- 左エリア
  - 入力PDFの選択欄
  - 先頭ページプレビュー
  - PDF のドラッグ&ドロップ領域
- 右上エリア
  - JPEG品質スライダー
  - 白背景合成に関する注記
- 右下エリア
  - 保存先フォルダ選択
  - 出力先サブフォルダ表示
  - 進捗バー
  - 実行ボタン

### 4.3 通常フロー

1. ホーム画面で「PDF → JPEG」を押します
2. 「PDFを選択」を押して、変換したいPDFを1つ選びます
3. 先頭ページプレビューが表示されることを確認します
4. JPEG品質を必要に応じて調整します
5. 「保存先を選択」を押して出力先フォルダを選びます
6. 出力先サブフォルダ表示を確認します
7. 「JPEG書き出しを実行」を押します
8. 完了ダイアログが表示されたら変換完了です

## 5. PDF→JPEG の出力ルール

### 5.1 入力と対象ページ

- 変換対象は単一PDFのみです
- 対象ページは全ページ固定です
- ページ範囲指定は初版ではできません

### 5.2 ファイル名ルール

出力ファイル名は固定で次の形式です。

- `PDF名_001.jpg`
- `PDF名_002.jpg`
- `PDF名_003.jpg`

`001` は3桁ゼロ埋めのページ番号です。

### 5.3 出力フォルダ構成

保存先に `C:\output` を選び、入力PDFが `report.pdf` の場合は次のように保存されます。

```text
C:\output\report\report_001.jpg
C:\output\report\report_002.jpg
C:\output\report\report_003.jpg
```

保存先フォルダ直下にJPEGが並ぶのではなく、PDF名のサブフォルダ配下へ出力されます。

### 5.4 同名JPEGがある場合

- 実行前に上書き確認ダイアログが表示されます
- ダイアログでは件数と代表例が表示されます
- 承認すると対象JPEGを一括で上書きします
- 承認しない場合は変換を開始しません

### 5.5 透明背景の扱い

PDFページに透明要素が含まれる場合、JPEG保存時は白背景へ合成されます。

これは初版仕様です。透明のまま保存することはできません。

## 6. PDF→JPEG 実行中の注意

- 実行中は入力PDF、保存先、品質を変更できません
- 実行中はホームへ戻れません
- 実行中にアプリを閉じようとすると確認ダイアログが表示されます
- 進捗欄には、何ページ中の何ページ目を処理しているかが表示されます

## 7. 初版で含まれない機能

次の機能は PDF→JPEG 初版には含まれていません。

- DPI指定
- サイズ変更
- PNG出力
- ページ範囲指定
- 命名テンプレート変更
- ZIP出力
- キャンセル機能

## 8. エラー時の対処

### 8.1 PDFが選べない / 読み込めない

- PDF以外のファイルを選んでいないか確認してください
- 壊れたPDFやページを持たないPDFは変換できません
- 他アプリで排他的に開かれている場合は閉じてから再実行してください

### 8.2 上書き確認が表示される

- 出力先サブフォルダに既存JPEGがあります
- 既存ファイルを残したい場合は別の保存先フォルダを選んでください

### 8.3 背景色が白になる

- 透明要素を含むページは白背景に合成する仕様です
- これは異常ではなく、初版の保存ルールです

### 8.4 トラブルシューティング一覧

| 症状                   | 主な原因                           | 対処                                 |
| ---------------------- | ---------------------------------- | ------------------------------------ |
| 実行ボタンが押せない   | PDF未選択 / 保存先未選択           | PDF と保存先フォルダの両方を設定する |
| 上書き確認が毎回出る   | 出力先サブフォルダに既存JPEGがある | 保存先を変えるか、上書きを承認する   |
| 期待した背景にならない | 透明要素が白背景へ合成される仕様   | 仕様どおりの結果か確認する           |
| 終了時に警告が出る     | 変換処理が進行中                   | 完了を待つか、内容を確認して終了する |

## 9. 手動確認の観点

PDF→JPEG 機能を手動確認するときは、次を順に確認してください。

1. ホーム画面から「PDF → JPEG」へ遷移できる
2. PDFを選ぶと先頭ページプレビューが表示される
3. 保存先を選ぶと出力先サブフォルダ表示が更新される
4. 実行すると `保存先/PDF名/PDF名_001.jpg` 形式で出力される
5. 既存JPEGがあると上書き確認ダイアログが表示される
6. 透明背景を含むページが白背景 JPEG として保存される
